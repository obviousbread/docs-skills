# Тиры должностей и их различия

ДИ собирается из общего каркаса (`structure.md`), но три аспекта меняются по тиру: **квалификация (1.2)**, **назначение и подчинение (1.4-1.5)**, **права (раздел 4)**, **цепочка СОГЛАСОВАНО**, **наличие пункта 5.7** про подчинённых.

Тиры в МVP — три:

| Код | Тир | Примеры |
|-----|-----|---------|
| `specialist` | Линейный сотрудник | специалист, ведущий специалист, главный специалист, инженер, экономист, юрисконсульт |
| `dept_head` | Начальник отдела | начальник отдела, начальник отделения |
| `unit_head` | Начальник управления | начальник управления, начальник департамента |

Расширение под `deputy_dg` (зам ГД) и `medical_*` — в будущем.

## Определение тира

Если пользователь не указал явно — выводить из названия должности:

- содержит «начальник управления», «начальник департамента» → `unit_head`
- содержит «начальник отдела», «начальник отделения», «заведующий отделом» → `dept_head`
- остальное → `specialist` (с подтверждением у пользователя при первой генерации в новой роли)

## Различия — матрица

### 1.2 Квалификация (опыт работы)

| Тир | Образование | ДПО | Опыт |
|-----|-------------|-----|------|
| `specialist` | Высшее (специалитет/магистратура) — направления из ПС | По профилю — «(при наличии)» | Без жёстких требований либо «без предъявления требований к стажу», по обобщённому уровню квалификации ПС |
| `dept_head` | Высшее (специалитет/магистратура) — направления из ПС | По профилю + управление проектами/менеджменту — «(при наличии)» | не менее 3 лет, из них на руководящих должностях не менее 1 года |
| `unit_head` | Высшее (специалитет/магистратура) — направления из ПС | По профилю + стратегическое управление — обязательно либо «(при наличии)» (зависит от ПС) | не менее 5 лет, из них на руководящих должностях не менее 3 лет |

Если ПС задаёт более строгие требования — использовать ПС.

### 1.4 Назначение и освобождение

Шаблон: «<Должность> назначается на должность и освобождается от должности приказом руководителя <short_name>, либо лица, исполняющего его обязанности, **по представлению <X>**, **согласованному с <Y>**, в установленном порядке в соответствии с действующим трудовым законодательством Российской Федерации.»

| Тир | Представление (X) | Согласование (Y) |
|-----|-------------------|------------------|
| `specialist` | начальника Отдела | начальником Управления |
| `dept_head` | начальника Управления | заместителем генерального директора по <направлению> |
| `unit_head` | заместителя генерального директора по <направлению> | — (без доп. согласования) |

Если у должности нет вышестоящего отдела или управления (СП напрямую подчинено ГД или замГД) — адаптировать цепочку и предупредить пользователя.

### 1.5 Подчинение

Шаблон: «<Должность> в своей работе подчиняется непосредственно <X>, в случае его отсутствия – <Y> либо лицу, исполняющему его обязанности.»

| Тир | Непосредственно (X) | При отсутствии (Y) |
|-----|---------------------|--------------------|
| `specialist` | начальнику Отдела | непосредственно начальнику Управления |
| `dept_head` | начальнику Управления | заместителю генерального директора по <направлению> |
| `unit_head` | заместителю генерального директора по <направлению> | генеральному директору |

### Раздел 4. Права — дополнительные пункты

| Право | `specialist` | `dept_head` | `unit_head` |
|-------|-------------|-------------|-------------|
| Базовые (предложения, информация, взаимодействие, повышение квалификации, ТК РФ) | да | да | да |
| Давать поручения работникам Отдела/Управления | нет | да | да |
| Инициировать рабочие встречи и совещания | нет | да | да |
| Представлять Отдел/Управление по поручению руководства | по поручению | да (Отдел) | да (Управление) |
| Инициировать проведение проверок в подразделениях и филиалах | нет | да | да |

### Раздел 5. Ответственность — пункт 5.7

| Тир | Включать пункт 5.7 («за неисполнение поручений руководства за работу подчинённых лиц») |
|-----|------|
| `specialist` | нет |
| `dept_head` | да |
| `unit_head` | да |

## Цепочка СОГЛАСОВАНО

Базовый минимум для всех тиров (всегда добавляется в любую ДИ). ФИО подставляются из `~/.docs-plugin/di/approvers.md`:

1. Начальник управления делами и кадровой политики — ключ `udkp_head`
2. Начальник отдела управления персоналом — ключ `hr_head`
3. Начальник юридического отдела — ключ `legal_head`
4. Профильный заместитель руководителя — ключ `curating_dgd` (выбирается по направлению деятельности СП)

Сверху на этот минимум накладывается **цепочка по тиру**:

| Тир | Дополнительно (перед общим минимумом) |
|-----|---------------------------------------|
| `specialist` | непосредственный руководитель (нач. Отдела) + начальник вышестоящего Управления |
| `dept_head` | начальник вышестоящего Управления |
| `unit_head` | — (профильный замГД уже в общем минимуме) |

**Полный порядок** (сверху вниз в таблице СОГЛАСОВАНО):

ФИО для всех ролей подставляются из `~/.docs-plugin/di/approvers.md`.

| Позиция | `specialist` | `dept_head` | `unit_head` |
|---------|--------------|-------------|-------------|
| 1 | `direct_supervisor` (нач. Отдела) | `unit_head` (нач. Управления) | `udkp_head` |
| 2 | `unit_head` (нач. Управления) | `curating_dgd` | `hr_head` |
| 3 | `curating_dgd` | `udkp_head` | `legal_head` |
| 4 | `udkp_head` | `hr_head` | `curating_dgd` |
| 5 | `hr_head` | `legal_head` | — |
| 6 | `legal_head` | — | — |

Логика: иерархия снизу вверх (свои), затем кадровая, юридическая, политическая.

## Когда тир не подходит

Если должность не укладывается в `specialist | dept_head | unit_head` (например, заместитель ГД, главный специалист с особым статусом, советник, главный врач филиала) — **не угадывать**, а уточнить у пользователя:

- какой логике подчинён назначения (приказ ГД, приказ вышестоящего ведомства, и т.п.);
- кто непосредственный руководитель;
- по какой цепочке согласуется;
- какой объём прав на распорядительные действия.

Записать выясненное либо как разовое решение, либо предложить дополнить `tier_rules.md` новым тиром.
