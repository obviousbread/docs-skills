# Структура должностной инструкции

Эталонная структура — на основе трёх согласованных образцов в `examples/`. При генерации **не отклоняться** от этого порядка и нумерации без явной просьбы пользователя.

## Общая раскладка документа

```
[правый блок УТВЕРЖДАЮ]
[пустая строка]

ДОЛЖНОСТНАЯ ИНСТРУКЦИЯ                                     ← bold, по центру
<полное наименование должности и учреждения>               ← по центру

1. Общие положения                                         ← bold, по центру
   1.1.  Настоящая должностная инструкция определяет ...
   1.2.  На должность ... назначается лицо, имеющее:
            – высшее образование – ...                     ← дашевые подпункты
            – дополнительное профессиональное образование – ...
            – опыт работы по направлению ...
   1.3.  К работе ... допускается лицо, прошедшее ...
   1.4.  ... назначается на должность и освобождается ...
   1.5.  ... в своей работе подчиняется непосредственно ...
   1.6.  ... должен знать:
            – Конституцию Российской Федерации;
            – Трудовой кодекс Российской Федерации ...
            – ...
   1.7.  В своей деятельности ... руководствуется:
            – законами и иными нормативными правовыми актами ...
            – ...

2. Трудовая функция                                        ← bold, по центру
   <Должность> осуществляет следующие функции:
   2.1.  ...
   2.2.  ...
   ...

3. Должностные обязанности                                 ← bold, по центру
   <Должность> выполняет следующие должностные обязанности:
   3.1.  ...
   ...
   3.N.  Соблюдает требования по охране труда:
            – правильно применять средства индивидуальной и коллективной защиты,
            – проходить обучение безопасным методам ...
            – немедленно сообщать ... о любой ситуации, угрожающей жизни и здоровью людей,
            – проходить обязательные предварительные и периодические медицинские осмотры,
            – другие требования охраны труда ...
   3.N+1. В случае возникновения чрезвычайной ситуации по решению руководства
          прибыть в <учреждение> в течение одного часа.

4. Права                                                   ← bold, по центру
   <Должность> имеет право:
   4.1.  ...
   ...
   4.N.  <Должность> пользуется всеми правами в соответствии с Трудовым кодексом
         Российской Федерации.

5. Ответственность                                         ← bold, по центру
   <Должность> несет ответственность за:
   5.1.  неисполнение или ненадлежащее исполнение должностных обязанностей,
         предусмотренных настоящей должностной инструкцией – в пределах,
         определенных действующим трудовым законодательством Российской Федерации;
   5.2.  причинение материального ущерба работодателю – ...;
   5.3.  разглашение конфиденциальной информации ...;
   5.4.  разглашение либо неправомерную обработку персональных данных;
   5.5.  некачественное ведение и организацию ведения документации ...;
   5.6.  за правонарушения, совершенные в процессе осуществления своей деятельности – ...
   5.7.  за неисполнение поручений руководства за работу подчиненных лиц.   ← только для руководителей

6. Заключительные положения                                ← bold, по центру
   6.1.  Ознакомление работника с должностной инструкцией ...
   6.2.  Должностная инструкция действует бессрочно. Условия пересмотра ...:
            – при изменении организационной структуры ...;
            – при пересмотре штатного расписания ...;
            – при появлении новых видов работ/услуг ...;
            – при внедрении новых технологий ...;
            – по иным случаям ...
   6.3.  Требования должностной инструкции являются обязательными для работника ...
   6.4.  Факт ознакомления работника с настоящей должностной инструкцией
         подтверждается подписью работника в экземпляре инструкции, хранящемся у работодателя.
   6.5.  Один экземпляр должностной инструкции хранится у работника, второй экземпляр
         у руководителя структурного подразделения Учреждения, третий – в отделе
         управления персоналом Учреждения.

ПРОЕКТ РАЗРАБОТАН:
[таблица 1 строки данных, 4 столбца: Должность | ФИО | Подпись | Дата;
 в каждой строке-данных под содержимым пустой параграф для визуального
 разреживания, как в листе согласования приказа]

СОГЛАСОВАНО:
[таблица N строк, 4 столбца: Должность | ФИО | Подпись | Дата;
 разреживание то же]

Лист ознакомления
С должностной инструкцией <род. падеж должности> <род. падеж учреждения>,
ознакомлен(а), один экземпляр получил(а) на руки и обязуюсь выполнять ее
и хранить на рабочем месте:
[таблица 23 строк, 3 столбца: ФИО | Подпись | Дата]
```

## Структура контента (агенту, чтобы знать формат `sections=[...]`)

- **Шесть разделов** с фиксированными заголовками (см. шаблон выше). Заголовок попадает в `section.title`.
- **Пункт раздела** = строка в `section.points`. Можно либо `"текст"` (станет `X.Y`), либо `(текст, level)` для редкого `X.Y.Z` (level=2).
- **Дашевые перечисления** внутри пункта (например, в 1.2 «образование», 1.6 «должен знать», 6.2 «условия пересмотра», блок ОТ в разделе 3) — через `{"text": "Вводная фраза:", "dash_items": ["элемент;", "...", "последний."]}`. Каждый элемент с маленькой буквы, заканчивается `;`; последний — `.`.
- **Не вписывать номера вручную** в текст пункта — нумерация делается Word через `_list_para`, пропусков и расхождений не бывает. В образцах юзера встречаются пропуски (`3.13 → 3.21`, `5.5 → 5.8`) — это ручные ошибки, повторять не надо.

## Что контролирует агент (генератор не правит)

Это правила про **смысл текста**, который агент кладёт в `section.points`. Генератор не перефразирует.

- **Кавычки**: только «ёлочки». Внутри них допустимы "лапки".
- **Слеши**: запрещены между кириллическими словами. Использовать запятые, «или», «и», скобки. Допустимо: `№ п/п`, номера документов, кодовые сокращения. Генератор предупреждает при нарушении.
- **Точка с запятой**: только в перечнях. В обычных предложениях — запятая или точка.

## Что генератор делает за агента (не проверять руками)

См. `helpers.md` для полного контракта. Кратко: шрифт, размер, межстрочный, отступы, выравнивание, цвет, нумерация (арабская, без пропусков), дашевый маркер `–`, авто-замена длинного тире `—` → `–`, авто-замена `ё` → `е`, гриф УТВЕРЖДАЮ, заголовок ДИ, таблицы подписей, лист ознакомления.

## Правила содержания по разделам

### 1. Общие положения (стандартный 7-пунктовый каркас)

- **1.1.** Что определяет инструкция. Шаблон:

  > Настоящая должностная инструкция определяет функциональные обязанности, права и ответственность <полное наименование должности> <full_name из org_details.md> (далее – <короткая должность>, <Отдел/Управление>, <short_name>, Учреждение).

- **1.2.** Квалификация. Состав:
  - высшее образование (специалитет/магистратура) с перечнем направлений из ПС;
  - ДПО (повышение квалификации/профпереподготовка) — «(при наличии)» если необязательно;
  - опыт работы. По тиру: специалист — без жёстких требований; нач. отдела — не менее 3 лет, из них на руководящих не менее 1 года; нач. упр. — не менее 5 лет, из них на руководящих не менее 3 лет.

- **1.3.** Медосмотры. Стандартный шаблон со ссылкой на ст. 220 ТК РФ и приказы Минздрава 29н/707н (или актуальные на момент генерации — проверять).

- **1.4.** Назначение и освобождение. Цепочка зависит от тира — см. `tier_rules.md`.

- **1.5.** Подчинение. Цепочка зависит от тира — см. `tier_rules.md`.

- **1.6.** Должен знать. Универсальный пул из `knowledge_base.md` + специфика из ПС/Положения о СП.

- **1.7.** Руководствуется в деятельности. Пул: законы РФ, Положение о СП (отдел и/или управление), ПВТР, ЛНА Учреждения, указания вышестоящих руководителей, настоящая инструкция.

### 2. Трудовая функция

Входит в раздел через шапку: «<Должность> осуществляет следующие функции:».

Пункты — это **обобщённые трудовые функции** (ОТФ) и **трудовые функции** (ТФ) из выбранного профстандарта, переформулированные под формат «осуществляет», «обеспечивает», «организует», «координирует». Дополнения из Положения о СП — допустимы, если расширяют, а не противоречат ПС.

Объём — обычно 7-12 пунктов.

### 3. Должностные обязанности

Шапка: «<Должность> выполняет следующие должностные обязанности:».

Пункты — это **трудовые действия** (ТД) из ПС, переформулированные через активные глаголы («организует», «обеспечивает», «координирует», «выполняет», «представляет», «подготавливает»).

Обязательные завершающие пункты (всегда последние два):

- Соблюдение охраны труда — дашевый блок из 5 пунктов (см. шаблон выше).
- При ЧС прибыть в учреждение в течение одного часа.

Объём — обычно 12-25 пунктов в зависимости от тира.

### 4. Права

Шапка: «<Должность> имеет право:».

Базовый минимум (для всех тиров):

- Вносить предложения по совершенствованию работы.
- Запрашивать и получать информационные материалы и НПА.
- Взаимодействовать со структурными подразделениями и филиалами.
- Принимать участие в совещаниях/конференциях по поручению руководства.
- Повышать квалификацию.
- Использовать все права, предусмотренные ТК РФ (последний пункт раздела).

Дополнительные права для руководителей — см. `tier_rules.md`.

### 5. Ответственность

Шапка: «<Должность> несет ответственность за:».

Стандартный набор пунктов 5.1-5.6 — для всех тиров (формулировки см. в шаблоне выше).

5.7 «за неисполнение поручений руководства за работу подчинённых лиц» — **только для руководителей** (нач. отдела, нач. упр., замы).

### 6. Заключительные положения

Универсальный 5-пунктовый блок — копируется без изменений (см. шаблон выше).

## Хвост документа

- **ПРОЕКТ РАЗРАБОТАН**: одна строка данных. Должность и ФИО — по умолчанию из `~/.docs-plugin/org_details.md` (`author_position` и `author_name_short`); при явном указании пользователем — его данные. В строке-данных под содержимым пустой параграф (визуальное разреживание, как в листе согласования приказа).
- **СОГЛАСОВАНО**: цепочка собирается по `tier_rules.md`; ФИО — из `~/.docs-plugin/di/approvers.md`. Всегда добавляются ключевые роли (`udkp_head`, `hr_head`, `legal_head`, `curating_dgd`) — фактические ФИО берутся из user-zone файла. Разреживание то же.
- **Лист ознакомления**: 23 пустых строки (стандарт), 3 столбца — ФИО / Подпись / Дата. В шапке — одна вступительная фраза с упоминанием полного наименования должности и учреждения.

## Чего НЕ делать

- Не добавлять «УТВЕРЖДАЮ» в склонениях «УТВЕРЖДЕНО/УТВЕРЖДЕНА» — для ДИ всегда «УТВЕРЖДАЮ» от первого лица руководителя.
- Не добавлять реквизиты «от ___ № ___» в гриф (в отличие от приложений к приказу). Гриф — пустые поля для дат.
- Не дублировать наименование должности в каждом пункте — после первого упоминания и `(далее – ...)` в 1.1 использовать сокращённое.
- Не вставлять блок состава комиссии или плана мероприятий — это другой тип документа.
- Не давать ссылку на «настоящий приказ» — ДИ не приказ, она утверждается отдельным грифом.
