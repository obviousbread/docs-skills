# Правила работы с профессиональными стандартами

ПС — основа разделов 1.2 «Квалификация», 1.6 «Должен знать», 2 «Трудовая функция», 3 «Должностные обязанности». Без ПС или с неправильным ПС вся ДИ юридически шаткая.

Перед любым `WebSearch` прочитай общий быстрый протокол `~/.docs-plugin/runtime/references/web-search.md` (в dev source: `references/web-search.md`). Он задаёт лимиты запросов, приоритет источников, stop rules, WebFetch и единый evidence JSON.

## Принцип

Никогда не выдумывать реквизиты ПС, формулировки ОТФ/ТФ/ТД. Всегда подтверждать через первичный источник:

1. **profstandart.rosmintrud.ru** — официальный реестр ПС Минтруда (приоритет).
2. **publication.pravo.gov.ru** — официальный портал правовой информации (для проверки приказа об утверждении).
3. **consultant.ru / garant.ru** — справочные системы (как вторичный источник, для удобства просмотра текста).

Отклонять выдачу LLM-кэша из памяти. Только верифицированные источники.

## Шаг 1. Поиск ПС

### Если пользователь указал ПС явно

Пользователь даёт код (`07.005`), номер приказа, название (`«Специалист по управлению персоналом»`) или ссылку.

Проверки:
1. ПС существует и **актуален** (не отменён, не заменён). Многие старые ПС заменены — например, 07.003 «Специалист по управлению персоналом» обновлялся.
2. ПС **применим к должности**. Если пользователь даёт ПС «Делопроизводитель» для должности «Начальник управления делами» — это несоответствие, спросить.
3. Зафиксировать реквизиты: код ПС, наименование ПС, номер и дата приказа Минтруда об утверждении, источник (URL).

### Если пользователь НЕ указал ПС

WebSearch по `web-search.md` на `profstandart.rosmintrud.ru` по ключевым словам должности и направления СП. Альтернатива — `consultant.ru` (там удобный поиск по названию), но финальные реквизиты лучше подтверждать первичным источником.

Что искать:
- основное название должности («начальник отдела стратегического развития»);
- ключевые функции из Положения о СП (если есть);
- отраслевую принадлежность (государственное управление, здравоохранение, ИТ, экономика, и т.п.).

Найти 1-3 кандидата. Для каждого собрать:
- код ПС (XX.YYY);
- наименование ПС;
- номер и дату приказа Минтруда об утверждении;
- перечень ОТФ с уровнями квалификации;
- релевантность должности (короткая мотивация).

Представить пользователю короткой таблицей и попросить выбрать. Не выбирать самостоятельно, если кандидатов больше одного с близкой релевантностью.

### Если ПС не найден

Бывает: для редких или новых должностей профстандарт ещё не утверждён. В этом случае:
1. Сообщить пользователю явно: «Профстандарт по этому профилю не найден в реестре Минтруда».
2. Предложить альтернативы:
   - использовать ПС смежной должности (например, для «Главный аналитик по мониторингу СМО» — ПС «Системный аналитик» 06.022);
   - использовать **квалификационные характеристики** из ЕКС (Единый квалификационный справочник должностей руководителей, специалистов и служащих, утв. постановлением Минтруда);
   - заполнить раздел 1.2 и 2-3 на основе Положения о СП без ссылки на ПС, отметив это в обосновании.
3. Не выдумывать «фальшивый» ПС.

## Шаг 2. Извлечение содержимого

### Для раздела 1.2 (квалификация)

Из ПС взять блок «Требования к образованию и обучению» и «Требования к опыту практической работы» из выбранного **уровня квалификации** (УК) — обычно у `specialist` это УК 6, у `dept_head` УК 7, у `unit_head` УК 7-8.

Перенести **дословно**, разбив на дашевые подпункты:
- образование (с перечнем направлений подготовки);
- ДПО (если требуется в ПС — обязательно; если нет — добавить «(при наличии)»);
- опыт работы.

Опыт работы по умолчанию — из тира (`tier_rules.md`). Если ПС задаёт более строгие требования — использовать ПС.

### Для раздела 1.6 (должен знать)

Собрать раздел «Необходимые знания» **всех применимых ТФ** в выбранной ОТФ. Дедуплицировать.

Не дублировать пункты, уже включённые в универсальное ядро (`knowledge_base.md`).

### Для раздела 2 (трудовая функция)

Каждая ТФ из выбранной ОТФ → один пункт раздела 2. Переформулировать через активные глаголы: «осуществляет», «обеспечивает», «организует», «координирует», «выполняет», «представляет», «подготавливает».

Не копировать заголовок ТФ дословно — он в ПС часто формулируется как существительное. Переписать в форме «<Должность> ...».

### Для раздела 3 (должностные обязанности)

Каждое **трудовое действие** (ТД) из применимых ТФ → один пункт раздела 3. Переформулировать через те же активные глаголы.

Если ТД дублирует пункт из раздела 2 (часто бывает) — раздел 2 формулирует общую функцию, раздел 3 — конкретные действия. Раздел 3 более детальный.

## Шаг 3. Валидация выбора ПС

После заполнения чернового документа — проверить:

- [ ] Все пункты раздела 2 имеют источник в ПС или Положении о СП.
- [ ] Все пункты раздела 3 имеют источник в ПС или Положении о СП.
- [ ] Уровень квалификации в 1.2 соответствует тиру должности.
- [ ] Опыт работы из ПС не противоречит правилу тира.
- [ ] В разделе 1.6 нет пунктов, которые объективно не относятся к должности (например, «методики проведения химико-токсикологических исследований» в ДИ начальника отдела закупок).

Если что-то не сходится — пересмотреть выбор ПС или явно зафиксировать отклонение.

## Шаг 4. Документирование выбора

В ответе пользователю (но **не в самом тексте ДИ**) указать:

- какой ПС выбран (код, название, приказ Минтруда);
- какие ОТФ и ТФ использованы;
- что добавлено сверх ПС из Положения о СП;
- что было опущено из ПС и почему.

Это нужно для последующей проверки — если кадровик спросит «откуда такие требования», должен быть ответ.

## Типичные проблемы

| Проблема | Что делать |
|----------|-----------|
| Найдено два релевантных ПС, оба применимы | Спросить пользователя; либо использовать оба (взять ОТФ из одного и блок «Знания» из обоих, без дублей) |
| ПС отменён, замены нет | Использовать ЕКС или Положение о СП, явно указать пользователю |
| ПС утверждён, но текст недоступен (не загружается) | WebSearch по `web-search.md` по альтернативным источникам (consultant, garant); если не найден текст — сообщить пользователю, не угадывать |
| ПС написан в общих формулировках, не покрывает специфику СП | Использовать ПС как базу + добавить специфику из Положения о СП, явно отметить расширения |
| ПС включает медицинскую составляющую (709н и т.п.) для немедицинской должности | Отфильтровать медицинские ТД, использовать только применимые |

## Что НЕ делать

- Не вставлять реквизиты ПС в раздел 1.2 в формате «в соответствии с профстандартом 07.003, утверждённым приказом Минтруда от ...». Реквизиты ПС в тексте ДИ как правило не указываются (но могут — по запросу пользователя).
- Не сокращать формулировки знаний из ПС до неузнаваемости. Допустима лёгкая адаптация под русскую грамматику.
- Не использовать ПС из других стран (бывает в LLM-кэше).
