# Хелперы generate.py — справочник

Что доступно агенту при сборке ДИ. Главная точка входа — `create_di`. Остальные хелперы — для возможных кастомных сценариев (если потребуется собрать нестандартный документ или дополнить лист ознакомления отдельно).

## Главная функция

### `create_di(position_full, position_full_genitive, sections, output_path, author=None, approvers=None, org=None, acquaintance_rows=23)`

Создаёт `.docx` целиком: гриф УТВЕРЖДАЮ → заголовок → 6 разделов → ПРОЕКТ РАЗРАБОТАН → СОГЛАСОВАНО → Лист ознакомления.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `position_full` | str | Полное наименование должности с учреждением — для подзаголовка после «ДОЛЖНОСТНАЯ ИНСТРУКЦИЯ» |
| `position_full_genitive` | str | Полное наименование должности в род. падеже (без учреждения) — для шапки Листа ознакомления |
| `sections` | list[dict] | Шесть разделов; формат — см. ниже |
| `output_path` | str \| None | Путь сохранения. Если None — `<output_dir_di>/ДИ <первое_слово>.docx` (output_dir_di из `~/.docs-plugin/org_details.md`); пустой output_dir_di — текущая директория. Если файл существует — суффикс ` 2`, ` 3`, ... |
| `author` | dict \| None | `{"position": ..., "name": ...}`. По умолчанию — `author_position` / `author_name_short` из `~/.docs-plugin/org_details.md` (алиасы `default_author_*` поддерживаются) |
| `approvers` | list[dict] | `[{"position": ..., "name": ...}, ...]` для СОГЛАСОВАНО |
| `org` | dict \| None | Реквизиты; если None — `_load_org_details()` |
| `acquaintance_rows` | int | Кол-во пустых строк в Листе ознакомления (по умолчанию 23) |

Возвращает фактический путь сохранения (с учётом коллизии).

## Формат `sections`

```python
sections = [
    {
        "title": "Общие положения",            # заголовок раздела (получит № автоматически)
        "intro": None,                          # str | None — текст без нумерации сразу после заголовка
        "points": [
            "Простой пункт станет 1.1.",
            ("Подпункт 1.1.1.", 2),             # (text, level) — для редкого X.Y.Z
            {                                    # пункт с дашевым подперечислением
                "text": "На должность ... назначается лицо, имеющее:",
                "dash_items": [
                    "высшее образование – ...;",
                    "опыт работы по направлению ...;",
                ],
                "depth": 1,                      # default 1; почти всегда оставлять
            },
        ],
    },
    {
        "title": "Трудовая функция",
        "intro": "Начальник Отдела осуществляет следующие функции:",  # без номера
        "points": [...],
    },
    # ...ещё 4 раздела
]
```

## Что генератор делает за агента (детерминированно)

| Аспект | Реализация |
|--------|------------|
| Times New Roman 14 pt везде | `style.font` + `_run(size=14)` |
| Межстрочный 1.0 | `pf.line_spacing = 1.0` в стиле и каждом параграфе |
| Поля A4 (верх 2.0, низ 2.0, лево 3.0, право 1.5 см) | в `create_di` при создании Document |
| Цвет — чёрный | дефолт Word (auto), нигде явно не задаём |
| Служебные метаданные python-docx очищены | общий `lib/docx_meta.py::new_document()` |
| Нумерация разделов 1, 2, 3 (арабская, жирная, по центру) | `_setup_di_numbering` ilvl=0, decimal, bold |
| Нумерация пунктов X.Y (без пропусков, авто-сброс при новой секции) | `_setup_di_numbering` ilvl=1, multilevel |
| Дашевый маркер `–` (en dash) | `_setup_dash_numbering` — bullet с U+2013 |
| Длинное тире `—` (U+2014) → среднее `–` (U+2013) | `_sanitize_text` в `_run` |
| Буква `ё` → `е` (`Ё` → `Е`) | `_sanitize_text` в `_run` |
| Гриф УТВЕРЖДАЮ в правом блоке | `_add_grif` |
| Заголовок «ДОЛЖНОСТНАЯ ИНСТРУКЦИЯ» жирным по центру + подзаголовок | `_add_title` |
| Таблицы ПРОЕКТ РАЗРАБОТАН и СОГЛАСОВАНО (4 столбца с пустой строкой под содержимым) | `_add_signoff_table` |
| Лист ознакомления отдельным разделом, 3 столбца, 23 строки | `_add_acquaintance_sheet` + `_add_section_break` |
| Имя файла без перезаписи (` 2`, ` 3`, ...) | `_resolve_output_path` |

## Что агент должен сам

| Проблема | Почему не автозаменяем |
|----------|------------------------|
| Кавычки `"..."` → `«...»` | Риск сломать вложенные `«ext "int" ext»`. Агент пишет правильные «ёлочки» сразу |
| Слеши между словами (`приказ/распоряжение`) | Контекстный выбор: запятая, «или», «и», скобки. Генератор предупреждает через `_warn_slash` |
| Точка с запятой в середине предложения | Не отличить от валидного перечня программно |
| Содержательное наполнение пунктов | ПС, Положение о СП, тир-правила — выбор агента |

## Низкоуровневые хелперы (если нужно собрать что-то нестандартное)

| Функция | Назначение |
|---------|-----------|
| `_run(para, text, ...)` | Run с TNR 14 + санитизация (длинное тире → среднее, ё → е, варн на /) |
| `_para(doc, text, ...)` | Параграф с нужными отступами, выравниванием, line_spacing=1.0 |
| `_cell_text(cell, text, ...)` | Одна строка в ячейку (vertical_align="center" по умолчанию) |
| `_cell_lines(cell, lines, ...)` | Несколько строк-параграфов в ячейку (без `\n`) |
| `_list_para(doc, text, level, num_id, ...)` | Пункт нумерованного списка через встроенную нумерацию Word |
| `_setup_di_numbering(doc)` | Многоуровневая нумерация ДИ (ilvl=0 → `N.`, ilvl=1 → `N.M.`) → numId |
| `_setup_dash_numbering(doc)` | Маркированный список с `–` → numId |
| `_remove_table_borders(table)` | Убрать все границы (для гриф-таблицы и блока подписей) |
| `_set_table_borders(table)` | Тонкие границы (для содержательных таблиц) |
| `_set_cell_margins_zero(table)` | Внутренние отступы ячеек = 0 |
| `_set_rows_no_break(table)` | Запретить разрыв строк таблицы между страницами |
| `_table_full_width_fixed(table, col_widths_dxa)` | Таблица 100% ширины с фиксированной разметкой столбцов (twips) |
| `_add_section_break(doc)` | Разрыв раздела с новой страницы (для Листа ознакомления) |
| `_load_org_details()` | Парсит `~/.docs-plugin/org_details.md` → dict реквизитов (с алиасами под имена полей docs-di) |

## Кастомный сценарий

Если нужно собрать ДИ без `create_di` (например, добавить дополнительный блок между разделами и подписями), импортируй низкоуровневые хелперы:

```python
from generate import (
    _new_document, _setup_di_numbering, _setup_dash_numbering,
    _add_grif, _add_title, _add_section, _add_signoff_table,
    _add_acquaintance_sheet, _load_org_details,
)

doc = _new_document()
# ... настройка стиля и полей (см. create_di)
di_id = _setup_di_numbering(doc)
dash_id = _setup_dash_numbering(doc)
org = _load_org_details()
_add_grif(doc, org)
_add_title(doc, "...")
for sec in sections:
    _add_section(doc, sec, di_id, dash_id)
# ... твой кастомный блок
_add_signoff_table(doc, "ПРОЕКТ РАЗРАБОТАН:", [author])
_add_signoff_table(doc, "СОГЛАСОВАНО:", approvers)
_add_acquaintance_sheet(doc, "...", org.get("short_name", ""))
doc.save("...")
```
