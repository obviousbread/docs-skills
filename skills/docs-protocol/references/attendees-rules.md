# Правила оформления участников протокола

## Формат ФИО

Единственный допустимый формат: **«Фамилия И.О.»** – фамилия, пробел, две заглавные буквы с точками.

- Правильно: `Алмазов А.А.`, `Бирюзова Б.Б.`
- Неправильно: `Алмазов Андрей Александрович`, `А.А. Алмазов`, `алмазов а.а.`

Внутри инициалов точки обязательны. Пробел между фамилией и инициалами обязателен. Пробела между инициалами нет.

## Должность

Должность пишется **строчными буквами** (lower case) – и у председателя, и у участников.

- Правильно: `заместитель директора по общим вопросам`
- Неправильно: `Заместитель директора по общим вопросам`

Код `_load_staff()` нормализует должности в `lower()` при загрузке из штатного списка; агент должен передавать их уже в нижнем регистре, чтобы совпадение было гарантировано.

## Председатель

Председатель задаётся отдельным словарём:

```python
chair = {
    "lastname":  "Алмазов",
    "initials":  "А.А.",
    "position":  "директор",  # строчные
}
```

В документе `_make_venue_chair_block` выводит строку:

```
Председатель:  Алмазов А.А. – директор
```

Метка «Председатель:» выделяется жирным. Должность вписывается после длинного тире (–) без орг-привязки в этой строке. Полная привязка к организации фигурирует в блоке подписи (`_make_signature_block`): там выводятся должность и краткое наименование организации.

## Фильтрация председателя из списка присутствовавших

`_make_attendees_table` **автоматически удаляет** председателя из списка участников, если он туда попал. Сравнение ведётся по паре `(lastname.lower(), initials.lower())`. При обнаружении совпадения функция выдаёт предупреждение:

```
Председатель «Алмазов А.А.» передан в attendees — отфильтрован.
```

Председатель не дублируется в таблице «Присутствовали». Передавать его в `attendees` не нужно; если всё же передать – предупреждение сигнализирует об избыточности, данные при этом корректны.

## Порядок участников

Порядок строк в таблице соответствует порядку элементов в переданном списке `attendees`. Сортировка по алфавиту не применяется. Обычная практика – передавать участников по убыванию должности (старшие первыми).

## Структура таблицы «Присутствовали»

Таблица без видимых границ, три колонки, ширина подстраивается под содержимое:

| Колонка | Содержимое | Пример |
|---------|-----------|--------|
| 1 | ФИО | `Бирюзов Б.Б.` |
| 2 | разделитель | `–` |
| 3 | Должность (строчные) | `начальник отдела кадров` |

Пример словаря участника:

```python
{"lastname": "Бирюзов", "initials": "Б.Б.", "position": "начальник отдела кадров"}
```

## Сверка ФИО со штатным списком

Перед генерацией `_verify_fios` сверяет председателя, каждого участника и каждого ответственного с `staff_file` (xlsx). При несовпадении выбрасывается `ValueError` с fuzzy-подсказками (до 5 близких совпадений). Передавайте ФИО строго в том виде, в котором они записаны в штатном списке.
