# Работа со списками сотрудников в протоколах совещаний

## Формат ФИО в разных частях документа

В протоколе действуют два разных порядка написания ФИО.

- **В теле документа** (строка «Председатель:», список «Присутствовали:», строки «Ответственные:», лист ознакомления) – **Фамилия И.О.** Пример: «Алмазов А.А.».
- **В блоке подписи** – **И.О. Фамилия**. Пример: «А.А. Алмазов». Функцию `_format_fio_initials_first` вызывает `_make_signature_block` автоматически; вручную передавать уже переставленное ФИО не нужно.

Полное ФИО (Фамилия Имя Отчество) в протоколе не используется, если пользователь явно не попросил иного.

## Чтение штатного списка: функция `_load_staff`

Файл берётся из поля `staff_file` в `~/.docs-plugin/org_details.md`. Функция открывает первый активный лист через `openpyxl` (режим `read_only=True, data_only=True`) и ищет заголовки в первой строке без учёта регистра.

| Колонка | Допустимые заголовки |
|---|---|
| ФИО | `фио`, `ф.и.о.`, `сотрудник` |
| Должность | `должность` |

Если колонка ФИО не найдена, функция возвращает пустой список. Если колонка «Должность» отсутствует, поле `position` остаётся пустой строкой.

Из значения ФИО функция извлекает фамилию и инициалы:

- При записи «Фамилия Имя Отчество» инициалы формируются как первые буквы второго и третьего слов с точками.
- При записи «Фамилия И.О.» (два слова, второе содержит точки) инициалы берутся как есть.

## Автоматическая сверка ФИО: функция `_verify_fios`

`create_protocol` вызывает `_verify_fios` перед сборкой документа. Проверяются три группы: председатель, каждый из присутствовавших, каждый ответственный из пунктов «РЕШИЛИ». Ответственный в `items[n]["responsible"]` должен быть передан в формате «Фамилия И.О.» – функция разбивает строку по пробелу и проверяет первые два слова.

Сверка ведётся по паре `(фамилия.lower(), инициалы.lower())` из загруженного штатного списка.

### Поведение при отсутствии staff_file

Если `staff_file` не задан, не существует или пуст, `_load_staff` возвращает пустой список, и `_verify_fios` поднимает `ValueError` с сообщением:

```
Не удалось загрузить штатный список для сверки ФИО.
Проверьте поле «staff_file» в ~/.docs-plugin/org_details.md
(файл должен существовать и содержать колонки «Фамилия», «Имя», «Отчество», «Должность»).
```

Передать пустой список явно (например, в тестах) можно так: `_verify_fios(..., staff=[])` – в этом случае сверка становится no-op без ошибки.

## Опечатка в ФИО: fuzzy-подсказки и правильное поведение агента

Если сотрудник не найден в штатном списке, функция запускает нечёткий поиск через `difflib.get_close_matches` (параметры: `n=5`, `cutoff=0.6`) и включает кандидатов в текст ошибки.

Пример сообщения при опечатке:

```
Присутствовавший «Бирюзов Б.Б.» не найден в штатном списке.
Возможно, имелось в виду: Бирюзов В.В., Бирюзова Б.Б.
```

**Алгоритм действий агента при получении такого ValueError:**

1. Показать пользователю полученное сообщение с кандидатами.
2. Задать один конкретный вопрос: какой из предложенных вариантов верный, или ввести правильное ФИО.
3. Дождаться подтверждения. Не исправлять ФИО автоматически.
4. После подтверждения повторить вызов `create_protocol` с исправленным значением.

Если кандидатов нет (ни один не преодолел порог 0.6), запросить у пользователя точное ФИО из штатного списка.

## Жёсткие правила

- Никогда не угадывать и не «исправлять» ФИО или должность без явного подтверждения пользователя.
- Сверку не делегировать субагенту: субагент не знает контекста документа и проверяет только тех, кого явно спросили.
- Должность подписанта (председателя в блоке подписи) брать из структуры `chair["position"]`, которую передаёт пользователь или заполняет скрипт из штатного списка. Поле `leader_title` из `org_details.md` относится к руководителю организации и к председателю совещания не применяется, если это разные люди.
