# Хелперы generate.py – справочник

Карта приватных функций генератора протоколов совещаний. Читать перед правкой `generate.py`.

---

## Загрузка конфига и штата

| Функция | Назначение | Когда менять |
|---|---|---|
| `_load_org_details()` | Читает `~/.docs-plugin/org_details.md`, возвращает словарь ключ-значение | При изменении формата `org_details.md` или добавлении новых реквизитов |
| `_load_staff()` | Загружает штатный список из `staff_file` (xlsx), возвращает `list[{lastname, initials, position}]`. При отсутствии файла возвращает `[]` | При изменении структуры xlsx-штатника или логики разбора колонок ФИО |
| `_build_org_config()` | Компонует итоговый конфиг из `_load_org_details()`: формирует `header_lines`, забирает `output_dir_protocol` и прочие поля | При добавлении нового поля из `org_details.md` в документ |
| `_verify_fios(chair, attendees, items, staff)` | Сверяет всех упомянутых лиц (председатель, присутствующие, ответственные) со штатным списком. При несовпадении выбрасывает `ValueError` с fuzzy-подсказками top-5 | При изменении порога нечёткого поиска или полей, которые проверяются |

## Форматтеры

| Функция | Назначение | Когда менять |
|---|---|---|
| `_warn_slash(text)` | Предупреждает при обнаружении косой черты между кириллическими словами; стандартные сокращения (п/п, ИНН/КПП) и дроби из исключений | При расширении списка допустимых исключений |
| `_format_ru_date(date_str)` | `«DD.MM.YYYY»` → `«D месяца YYYY г.»` без ведущего нуля | При изменении требований к формату даты в шапке |
| `_format_fio_initials_first(fio)` | `«Фамилия И.О.»` → `«И.О. Фамилия»`. Если порядок уже верный – возвращает без изменений | При смене порядка инициалов в подписи или секретаре |

## Примитивы разметки документа

| Функция | Назначение | Когда менять |
|---|---|---|
| `_run(para, text, ...)` | Добавляет run с Times New Roman, языком `ru-RU` и конвертацией `—` → `–`. Символ `\n` превращается в мягкий разрыв строки (`<w:br/>`) | При смене базового шрифта, кегля по умолчанию или языковых атрибутов |
| `_para(doc, text, ...)` | Обычный параграф с выбором выравнивания, красной строки и межстрочного интервала | При изменении типовых параграфных стилей документа |
| `_dash_para(doc, text, ...)` | Буллет-пункт с «–»: отступ слева 1.25 см, без красной строки, без интервала до и после | При изменении оформления маркированных подпунктов |
| `_cell_text(cell, text, ...)` | Одна строка в ячейку таблицы с вертикальным выравниванием | При изменении типового оформления ячеек |
| `_cell_lines(cell, lines, ...)` | Несколько строк в ячейку как отдельные параграфы (альтернатива `\n` там, где нужны независимые `pPr`) | При необходимости разных стилей на строку внутри ячейки |

## Таблицы: границы и автоподбор

| Функция | Назначение | Когда менять |
|---|---|---|
| `_set_cell_margins_zero(table)` | Обнуляет внутренние отступы всех ячеек через `w:tblCellMar` | При изменении типовых отступов ячеек |
| `_remove_table_borders(table)` | Убирает все границы таблицы (layout-таблицы: подпись, дата и номер) | Редко; при смене политики видимости границ |
| `_set_table_borders(table)` | Выставляет тонкие одиночные границы `0.5pt` (содержательные таблицы: лист ознакомления) | При изменении толщины или цвета линий |
| `_autofit_contents(table)` | AutoFit Contents: ширина таблицы 100%, столбцы по содержимому. Применять ко всем содержательным таблицам; не применять к layout-таблицам | При изменении модели ширины таблиц |
| `_set_rows_no_break(table)` | Запрет переноса строк между страницами (`cantSplit`) | При изменении политики разрывов страниц в таблицах |

## Блоки документа

| Функция | Назначение | Когда менять |
|---|---|---|
| `_make_header_block(doc, org)` | Шапка: строки из `header_lines` жирным по центру + адрес и телефон/факс курсивом 10pt | При изменении состава или порядка строк шапки |
| `_make_title_block(doc, subtype)` | «ПРОТОКОЛ» + подтип: две строки жирным по центру | При смене формата заголовка |
| `_make_date_number_table(doc, doc_date, doc_number)` | Таблица 1×2 без границ: дата слева, «№ …» справа. Если `doc_number` пустой – ставит «№ _________» | При изменении формата строки даты и номера |
| `_make_venue_chair_block(doc, venue, chair)` | Параграфы «Место проведения:» и «Председатель:» с жирными метками | При изменении состава или формата метаданных совещания |
| `_make_attendees_table(doc, attendees, chair)` | Параграф «Присутствовали:» + 3-столбцовая таблица: ФИО – должность. Автоматически фильтрует председателя из списка | При изменении состава столбцов или логики дедупликации |
| `_make_resolved_block(doc, items)` | Метка «РЕШИЛИ:» + цикл по пунктам через `_make_resolved_item` | При изменении заголовка блока решений |
| `_make_resolved_item(doc, item, num_id, dash_id)` | Один пункт «РЕШИЛИ»: нумерованный параграф + «Ответственные:» + «Срок:» + en-dash подпункты. Возвращает актуальный `dash_id` | При изменении состава полей пункта или отступов |
| `_make_signature_block(doc, chair, org)` | Таблица подписи 1×2: должность + организация слева, «И.О. Фамилия» справа | При изменении формата блока подписи |
| `_make_secretary_block(doc, secretary)` | Два параграфа секретаря: метка «Секретарь» и строка «должность → tab → И.О. Фамилия» | При изменении оформления секретарской строки |
| `_build_notify_persons(chair, attendees, items, staff)` | Строит список для листа ознакомления: объединение присутствующих и ответственных без председателя, дедупликация, сортировка А-Я | При изменении правил включения лиц в лист ознакомления |
| `_make_notify_sheet(doc, persons)` | Новая страница: заголовок «Лист ознакомления» + 4-столбцовая таблица с границами | При изменении состава столбцов листа ознакомления |

## Word-нумерация (numId не хардкодится)

Все три функции вычисляют свободный `numId` динамически через `_next_num_id`, поэтому документы с уже существующими списками не ломаются. `abstractNum` всегда вставляется строго перед первым `w:num` – требование порядка OOXML (ISO 29500-1 §17.9).

| Функция | Назначение | Когда менять |
|---|---|---|
| `_next_num_id(doc)` | Возвращает следующий свободный `numId` в `numbering.xml`. При отсутствии секции нумерации создаёт её через временный список | При смене логики инициализации `numbering_part` |
| `_setup_protocol_numbering(doc)` | Регистрирует decimal-нумерацию `%1.` (отступ left=1069, hanging=360) для пунктов «РЕШИЛИ». Возвращает динамический `numId` | При изменении формата или отступов нумерованных пунктов |
| `_setup_dash_numbering(doc)` | En-dash bullet для подпунктов «РЕШИЛИ» (отступ left=1429, hanging=360). Возвращает динамический `numId` | При изменении символа или отступов подпунктов |
| `_attach_numbering(para, num_id, ilvl)` | Присоединяет `numPr` к параграфу по `num_id` и `ilvl` | При добавлении вложенных уровней нумерации |

## Публичный API

| Функция | Назначение |
|---|---|
| `create_protocol(subtype, chair, attendees, items, ...)` | Собирает полный документ: загружает конфиг и штат, вызывает сверку ФИО, последовательно строит все блоки, сохраняет `.docx`, логирует генерацию. Возвращает путь к файлу. |
