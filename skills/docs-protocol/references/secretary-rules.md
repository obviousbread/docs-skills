# Блок секретаря

## Поведение по умолчанию

Блок секретаря **не добавляется** по умолчанию. Добавить его можно только при явном согласии пользователя – например, ответ «да» на вопрос «Укажите секретаря совещания?» или самостоятельное упоминание секретаря в запросе.

Если пользователь не упоминает секретаря, не уточнять и не предлагать.

## Что собрать от пользователя

Когда пользователь подтвердил наличие секретаря, запросить:

- ФИО секретаря (фамилия + инициалы или полное ФИО);
- должность секретаря (строчными, например «ведущий специалист отдела кадров»).

## Формат блока в документе

Блок состоит из двух параграфов, добавляемых после блока подписи председателя:

1. **Метка** «Секретарь» – жирным шрифтом, выравнивание по левому краю.
2. **Строка реквизита** – должность, затем символ табуляции, затем ФИО в формате «И.О. Фамилия» (инициалы первыми).

Технически: `_make_secretary_block(doc, secretary)`, где `secretary` – словарь с ключами `lastname`, `initials`, `position`. Функция возвращает `None` сразу, если `secretary` равен `None` или пустому значению.

Преобразование ФИО: «Берёзов Д.Д.» → «Д.Д. Берёзов» выполняет `_format_fio_initials_first`.

## Секретарь и лист ознакомления

Секретарь **не включается** в лист ознакомления. Лист ознакомления содержит подписантов протокола: присутствовавших и ответственных за поручения. Секретарь ведёт протокол, но подписантом не является.

## Пример

Входные данные:

```python
secretary = {
    "lastname": "Деревьева",
    "initials": "А.С.",
    "position": "специалист отдела документационного обеспечения",
}
```

Результат в документе:

```
Секретарь
специалист отдела документационного обеспечения    А.С. Деревьева
```

Слово «Секретарь» выводится жирным. Между должностью и ФИО стоит символ табуляции (не пробелы).
