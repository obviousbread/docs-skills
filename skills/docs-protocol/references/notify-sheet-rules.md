# Лист ознакомления: правила формирования

Источник истины: `_build_notify_persons` и `_make_notify_sheet` в `generate.py`.

---

## Состав: кто попадает в лист

Лист ознакомления формируется как объединение двух групп:

1. **Присутствовавшие** (`attendees`) – все участники, переданные в `create_protocol`.
2. **Ответственные** – каждое ФИО из поля `responsible` каждого пункта повестки (`items[*].responsible`).

Дубликаты устраняются до составления таблицы.

---

## Ключ дедупликации

Каждый человек идентифицируется парой `(фамилия, инициалы)` в нижнем регистре:

```
k = (lastname.lower(), initials.lower())
```

Если одно и то же лицо присутствует в attendees и одновременно указано ответственным в каком-либо пункте, в лист оно попадает **ровно один раз**.

---

## Исключение председателя

Председатель (`chair`) в лист ознакомления **не включается**. Он является подписантом протокола и в лист не дублируется.

Проверка выполняется по тому же ключу `(фамилия, инициалы)` в нижнем регистре:

```python
chair_key = (chair["lastname"].lower(), chair["initials"].lower())
# лица с k == chair_key пропускаются
```

---

## Порядок сортировки

Список отсортирован **по алфавиту А-Я по фамилии** (точнее: по ключу `(фамилия, инициалы)` через встроенный `sorted(keys)`).

> Это намеренное отклонение от иерархического порядка образца от 12.05.2026. Иерархию не восстанавливать.

---

## Разрыв страницы

Лист ознакомления начинается на **отдельной странице**. Перед заголовком вставляется `doc.add_page_break()` – разрыв страницы в терминах `python-docx`.

---

## Структура страницы

После разрыва размещаются:

1. Параграф «Лист ознакомления» – центр, жирный, 14 pt.
2. Параграф «с протоколом совещания» – центр, обычный, 14 pt.
3. Таблица (см. ниже).

---

## Таблица: 4 колонки

| Колонка | Заголовок | Содержимое строки |
|---------|-----------|-------------------|
| 1 | **№ п/п** | порядковый номер (1, 2, 3…) |
| 2 | **Должность** | должность из штатного списка (`position`) |
| 3 | **Ф.И.О.** | `«Фамилия И.О.»` (фамилия, пробел, инициалы) |
| 4 | **Подпись** | пустая ячейка для собственноручной подписи |

Заголовки строки 0 выводятся жирным (`bold=True`). Нумерация строк начинается с 1. Таблица имеет видимые границы (`_set_table_borders`) и режим AutoFit contents (`_autofit_contents`).

---

## Что происходит, если лицо не найдено в штате

Если ключ `(фамилия, инициалы)` не найден в `staff_by_key`, запись всё равно добавляется в лист с пустой должностью:

```python
{"lastname": k[0].title(), "initials": k[1].upper(), "position": ""}
```

Это запасной вариант на случай расхождения данных. В штатной работе сверка через `_verify_fios` должна отловить опечатку раньше.

---

## Пример входных данных и ожидаемый результат

```python
chair      = {"lastname": "Громов", "initials": "А.В.", "position": "директор"}
attendees  = [
    {"lastname": "Зайцева", "initials": "Н.П.", "position": "начальник отдела"},
    {"lastname": "Мартынов", "initials": "С.И.", "position": "ведущий специалист"},
]
items = [
    {
        "text": "Утвердить план мероприятий.",
        "responsible": ["Зайцева Н.П.", "Белова К.Р."],
        "deadline": "01.09.2026",
    }
]
```

Ожидаемый лист (алфавитный порядок по фамилии, председатель Громов исключён):

| № п/п | Должность | Ф.И.О. | Подпись |
|-------|-----------|--------|---------|
| 1 | специалист | Белова К.Р. | |
| 2 | начальник отдела | Зайцева Н.П. | |
| 3 | ведущий специалист | Мартынов С.И. | |

> Все ФИО в примере вымышлены.
