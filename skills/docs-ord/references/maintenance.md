# Правило согласованности: generate.py ↔ SKILL.md ↔ review-checklist.md

**Любое изменение в `generate.py` должно быть отражено в SKILL.md и `review-checklist.md`.**

После каждой правки `generate.py` проверь и при необходимости обнови:

| Что изменилось | Где обновить |
|----------------|-------------|
| Новая функция-хелпер | `references/helpers.md` |
| Изменилось поведение по умолчанию (`size`, `align`, `vertical_align` и т.п.) | `references/helpers.md` + SKILL.md раздел «Стиль текста» |
| Изменился API `create_ord()` (новый параметр, новый формат) | `references/helpers.md` + `references/usage-examples.md` |
| Изменилась структура разделов документа | SKILL.md шаг 5 «Описать структуру пользователю» |
| Новое ограничение или запрет (no `\n`, no `size=12` и т.п.) | SKILL.md раздел «Стиль текста» |
| Изменился порядок разделов / завершающих пунктов | SKILL.md шаг 5 + reference-файлы по теме |
| Изменился любой инвариант формата (поля, шрифт, таблицы блоков, разрывы разделов, нумерация, столбцы листов) | `references/review-checklist.md` — режим ревью сверяет документ именно по нему |

Это правило работает в обе стороны: если в SKILL.md появляется новое форматное требование – оно должно быть реализовано в `generate.py` и отражено в `review-checklist.md`. Чек-лист ревью обязан соответствовать тому, что реально генерирует `generate.py`, иначе ревью будет ловить ложные расхождения.
