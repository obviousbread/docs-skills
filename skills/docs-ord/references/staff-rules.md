# Работа со списками сотрудников

## Структура учреждения

Специфика структуры организации, правила именования подразделений и филиалов, типовые формулировки обращений описаны в поле `org_structure_notes` раздела `## Структура учреждения` файла `~/.docs-plugin/org_details.md`. Прочитай этот раздел перед формированием обращений и списков ознакомления.

## Единый источник правды

Путь к файлу (или директории с файлами) берется из поля `staff_file` в `~/.docs-plugin/org_details.md`. Если поле пустое - спроси пользователя, где находится файл, и предложи сохранить путь через `docs-init`.

### Если staff_file указывает на директорию

Применяй следующий алгоритм выбора файла:

1. Найти все поддиректории с датами (формат ДД.ММ.ГГГГ или ГГГГ-ММ-ДД). Выбрать ту, у которой дата максимальная (самая свежая).
2. Внутри выбранной директории искать файл, в имени которого содержится слово «сводный» (регистронезависимо). Если такой файл есть – использовать его.
3. Если сводного файла нет – использовать файл с наибольшим охватом (например, файл без указания конкретного подразделения в имени).

```python
import os, re
from datetime import datetime

def find_staff_file(staff_dir):
    staff_dir = os.path.expanduser(staff_dir)
    # Найти поддиректории с датами
    date_dirs = []
    for entry in os.scandir(staff_dir):
        if entry.is_dir():
            for fmt in ("%d.%m.%Y", "%Y-%m-%d"):
                try:
                    date_dirs.append((datetime.strptime(entry.name, fmt), entry.path))
                    break
                except ValueError:
                    pass
    if date_dirs:
        _, latest_dir = max(date_dirs, key=lambda x: x[0])
    else:
        latest_dir = staff_dir
    # Искать сводный файл
    for fname in os.listdir(latest_dir):
        if "сводный" in fname.lower() and fname.endswith((".xlsx", ".xls", ".csv")):
            return os.path.join(latest_dir, fname)
    # Fallback – первый xlsx/xls
    for fname in os.listdir(latest_dir):
        if fname.endswith((".xlsx", ".xls", ".csv")):
            return os.path.join(latest_dir, fname)
    return None
```

### Если staff_file указывает на файл

Использовать файл напрямую.

Формат сводного файла (6 столбцов):

| № | Сотрудник | Дата рождения | Должность | Подразделение | Филиал |

Если сводный файл отсутствует, допускается использование отдельных файлов филиалов (.csv/.xlsx), но приоритет всегда у сводного файла в самой свежей поддиректории.

## Жесткие правила

- Обязательная проверка перед каждым использованием ФИО и должности: всегда сверяй ФИО и должность сотрудника со списком сотрудников.
- Никаких домыслов (если напрямую не допускается пользователем), исправлений ФИО или должностей без точного соответствия таблице.
- Если ФИО или должность отсутствуют в таблице - сообщи пользователю и попроси уточнений.
- Запрещено использовать любую должность или ФИО без проверки их точного наличия в списке сотрудников.
- Можешь сверять по фамилии. Например: «[Фамилия И.О.]» → «[Фамилия Имя Отчество]», «[Фамилия]» → «[Фамилия Имя Отчество]».
- **Дубли в списке:** если сотрудник встречается несколько раз с разными должностями, использовать должность с наивысшим иерархическим уровнем. Начальник отдела > старший специалист; заместитель генерального директора > начальник управления и т.п. Вышестоящая должность является основной.
- **Подписант (руководитель организации):** должность и статус (в т.ч. «и.о.») берется из `~/.docs-plugin/org_details.md` (поля `leader_title`, `leader_name_nom`), а не из списка сотрудников. Список не всегда отражает актуальный статус исполняющего обязанности.

## Иерархия должностей и выбор старшего руководителя

При выборе сотрудника для текста документа, листа ознакомления или иных блоков недостаточно брать первое буквальное совпадение по подразделению. Сначала нужно определить **управленческий контур**, а затем выбрать старшего по иерархии внутри этого контура.

### Базовая иерархия

По умолчанию использовать следующий порядок старшинства:

1. Генеральный директор, и.о. генерального директора
2. Заместитель генерального директора
3. Руководитель самостоятельного крупного контура:
   - начальник управления
   - главный бухгалтер
   - директор центра
   - директор ЛРЦ
   - директор филиала
4. Руководитель подчиненного подразделения:
   - начальник отдела
   - заведующий канцелярией
   - заведующий кабинетом
   - иной руководитель самостоятельной службы
5. Заместитель руководителя подразделения:
   - заместитель начальника отдела
   - заместитель директора ЛРЦ
   - заместитель директора центра
   - заместитель директора филиала
6. Ведущий, старший специалист
7. Специалист и иные неруководящие должности

### Правило управленческого контура

Сначала определить, к какому контуру относится упоминание в документе:

- Если документ касается центральной функции учреждения, выбирать старшего руководителя центрального контура.
- Если документ касается конкретной площадки, выбирать директора этой площадки.
- Если документ касается всех площадок, включать директоров всех соответствующих площадок.
- Если документ касается отдела, который входит в состав управления, по умолчанию выбирать начальника управления, а не начальника отдела.
- Начальника отдела выбирать только тогда, когда:
  1. у отдела нет вышестоящего управления;
  2. документ адресован именно этому уровню;
  3. по смыслу нужен не руководитель контура, а руководитель конкретного отдела.

### Примеры применения

- Отдел информационных технологий Управления информационных и телемедицинских технологий -> для руководителя контура брать начальника управления, а не начальника отдела.
- Канцелярия -> брать заведующего канцелярией, если над ней нет отдельного управления.
- [Название центра] -> брать директора центра.
- [Название ЛРЦ] -> брать директора ЛРЦ.
- [Название филиала] -> брать директора филиала.

### Жесткое правило

Нельзя выбирать сотрудника только по наиболее точному текстовому совпадению названия подразделения. Сначала определяется управленческий контур, затем внутри него выбирается должность с наивысшим иерархическим уровнем.

### Связь с листами согласования и ознакомления

- Для листа ознакомления включать старшего релевантного руководителя затронутого контура, если он не попадает в лист согласования.
- Для листа согласования поднимать уровень до заместителя генерального директора или иного обязательного согласующего по правилам `approval-rules.md`.

### Алгоритм

1. Определи затронутый контур.
2. Найди верхний релевантный уровень управления внутри него.
3. Проверь, не входит ли найденное подразделение в более крупное управление.
4. Выбери старшего по иерархии.
5. Только после этого решай, идет ли сотрудник в согласование или в ознакомление.

## Формат ФИО в документе

- В теле документа, странице согласования, листе ознакомления: **Фамилия И.О.** (например, «[Фамилия И.О.]»).
- В блоке подписи руководителя: **И.О. Фамилия** (например, «[И.О. Фамилия]»).
- Полное ФИО (Фамилия Имя Отчество) - в документе не используется, если пользователь явно не уточнил иное.

## Сверка через openpyxl (самостоятельно, без субагента)

Проверку сотрудников НЕ делегировать субагенту - субагент не знает содержания документа, ищет только тех, кого явно спросили, и пропускает всех остальных. Делать самостоятельно:

```python
python3 -c "
import openpyxl, os
staff_file = os.path.expanduser('<staff_file из ~/.docs-plugin/org_details.md>')
wb = openpyxl.load_workbook(staff_file, read_only=True, data_only=True)
ws = wb.active
for row in ws.iter_rows(values_only=True):
    if row[4] and 'ключевое слово' in str(row[4]).lower():
        print(row)
wb.close()
"
```

Логика: определить, какие подразделения затронуты документом → найти всех их сотрудников → выбрать старшего по иерархии (начальник отдела > ведущий специалист > специалист) → добавить в документ и лист ознакомления.
