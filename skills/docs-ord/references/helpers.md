# Хелперы generate.py – справочник

## Таблица хелперов

| Функция | Назначение |
|---------|-----------|
| `_para(doc, text, ...)` | Обычный параграф |
| `_dash_para(doc, text, ...)` | Буллет с «–»: отступ слева 1.25 см, без красной строки |
| `_run(para, text, ...)` | Run с шрифтом Times New Roman. Санитизация: длинное тире -> среднее; предупреждение при «/» между кириллическими словами |
| `_cell_text(cell, text, ...)` | Одна строка в ячейку. По умолчанию: 14pt, `vertical_align="center"` |
| `_cell_lines(cell, lines, ...)` | Несколько строк в ячейку как отдельные параграфы (без `\n`). По умолчанию: 14pt, `vertical_align="center"` |
| `_list_para(doc, text, level, num_id, ...)` | Пункт списка с встроенной нумерацией Word |
| `_setup_ord_numbering(doc)` | Инициализировать нумерацию в документе -> numId |
| `_add_num_instance(doc)` | Новый независимый счётчик (для приложений) -> numId. Счётчик всегда начинается с 1, даже в Compatibility Mode -- см. примечание ниже |
| `_setup_polozheniye_numbering(doc)` | Многоуровневый список для Положения: ilvl=0 → Roman (I/II/III), ilvl=1 → decimal+isLgl (1.1/2.3). `isLgl` конвертирует Roman→Arabic в `%1.%2.` |
| `add_polozheniye(doc, sections)` | Положение: разделы I/II/III (ilvl=0) + пункты 1.1/2.3 (ilvl=1). Первая строка 1.25 см, остальной текст без отступа. |
| `_utverzhden_form(title)` | Автоматическое склонение «УТВЕРЖДЕН/О/А/Ы» по первому слову заголовка приложения |
| `_add_commission_table(doc, members)` | Таблица состава комиссии без границ. members: `[{"fio": "Фам\nИмя Отч", "position": "должность", "role": "председатель"|None}]` |
| `_add_attachment_block(doc, att, n, total, org, dt)` | Один блок приложения (гриф + содержимое): вызывается в цикле для каждого приложения |
| `_set_rows_no_break(table)` | Запрет разрыва строк между страницами (cantSplit). Вызывать на всех содержательных таблицах. |
| `_add_section_break(doc, landscape=False)` | Разрыв раздела с новой страницы + нумерация страниц. `landscape=True` – альбомный формат (A4 29.7×21 см). |
| `_remove_table_borders(table)` | Убрать все границы |
| `_set_table_borders(table)` | Тонкие границы (single, 0.5pt) |
| `_autofit_contents(table)` | AutoFit Contents: таблица на 100% ширины листа, столбцы подстраиваются под содержимое. Применять ко всем содержательным таблицам (перечни, листы ознакомления и т.д.). Не применять к layout-таблицам (подпись, дата и номер). |

## Нумерация в приложениях – Compatibility Mode

**Проблема (решена):** Word в Compatibility Mode игнорировал `startOverride` для второго и последующих списков, продолжая сквозную нумерацию (например, 9. вместо 1. в Приложении).

**Причина:** нарушение порядка элементов OOXML – все `w:abstractNum` обязаны располагаться до всех `w:num` (ISO/IEC 29500-1 §17.9). Если `abstractNum` вставлялся после уже существующих `w:num`, Word при последовательном разборе XML не находил определение вовремя и наследовал кешированный счётчик.

**Реализованный fix (в generate.py):**
- `abstractNum` вставляется через `root.insert(first_num_index, ...)` – строго перед первым `w:num`
- Каждый `abstractNum` получает уникальные `w:nsid` и `w:tmpl` (производные от `abs_id`) – предотвращает объединение одинаково выглядящих определений

Это поведение встроено в `_setup_ord_numbering` и `_add_num_instance` – при использовании этих функций ничего дополнительно делать не нужно.
