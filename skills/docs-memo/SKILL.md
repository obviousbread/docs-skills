# docs-memo — Служебные записки

Генерирует служебные записки в формате .docx. Реквизиты организации загружаются из `~/.docs-plugin/org_details.md`.

## 0. Карта references

| Файл | Что содержит | Когда читать |
|------|-------------|-------------|
| `speech-patterns.md` | Формальные речевые обороты: просьбы, уведомления, обоснования, ссылки на документы | Перед формулированием текста |
| `examples.md` | Примеры реальных служебок с темами и формулировками | При подборе формулировок для нестандартных тем |
| `~/.docs-plugin/org_details.md` | Адресат, автор, подписант по умолчанию | До начала работы |
| `~/.docs-plugin/memo/examples/memo.md` | Полные тела реальных служебных записок (добавлены docs-finetune) | Если существует |

## Структура документа

```
[пустой левый блок]  |  Кому (должность + ФИО дат.)
                     |  от кого (должность + ФИО род.)

             Служебная записка       (жирный, по центру)

    [Тело: абзацы с красной строкой 1.25 см, по ширине]

                                     ФИО подписанта
                                     ДД.ММ.ГГГГ
```

**Форматирование:** Times New Roman 14pt, A4, поля: верх 2 / лево 3 / право 1.5 / низ 1 см.

## Алгоритм работы

> **Gemini CLI:** `~/.docs-plugin/org_details.md` не подгружается автоматически. На шаге 1 прочитай файл явно. Используй native Gemini tools. Если файла нет — сообщи пользователю и предложи запустить docs-init.
>
> **Codex:** `~/.docs-plugin/org_details.md` не подгружается автоматически. На шаге 1 прочитай файл явно. Используй native Codex tools. Если файла нет — сообщи пользователю и предложи запустить docs-init.

1. Прочитать `~/.docs-plugin/org_details.md` (адресат, автор, подписант по умолчанию).
2. Прочитать `references/speech-patterns.md`.
3. Уточнить суть и цель (только если не указано в запросе).
4. Если нужна верификация НПА или исследование - запустить субагента (см. ниже). Дождаться результата.
5. Сформулировать текст, сохраняя официально-деловой стиль.
6. Использовать дату из запроса; если не указана - дефолт сегодня.
7. Запустить `create_sluzhebka()` и сохранить .docx.
8. Сообщить пользователю путь к файлу.

## Ситуативная верификация

**Субагентов не нужно**, если служебка - простая просьба, уведомление или пояснение.

**Субагент `npa-verifier`** - когда служебка ссылается на НПА:

```
Для каждого НПА из списка: <список или тема>
Проверь через WebSearch реквизиты и актуальность (consultant.ru, garant.ru, pravo.gov.ru).
Верни JSON-массив: [{"official_name": "...", "number": "...", "date": "...", "status": "актуален"|"отменен"|"изменен", "notes": "..."}]
Не придумывай реквизиты.
```

> **Claude Code:** запусти субагентом как описано выше.
>
> **Gemini CLI:** субагент недоступен. Выполни inline: WebSearch по каждому НПА на consultant.ru / garant.ru / pravo.gov.ru. Кодексы, Конституция, ГОСТы, СанПиНы - без реквизитов «от ... №...».
>
> **Codex:** если в текущей сессии доступен agent/subagent tool, можно делегировать проверку. Иначе сообщи пользователю `subagents unavailable in this session, continuing inline` и выполни ту же inline-проверку.

**Субагент `research-subagent`** - когда нужно проверить факты из открытых источников:

```
Тема: <тема из контекста служебки>
Задача: собрать актуальные данные, факты или статистику для обоснования позиции.
Источники: официальные сайты ведомств, правовые базы, профильные ресурсы.
Верни: {"summary": "краткий синтез для включения в служебку", "sources": ["url1", "url2", ...]}
```

> **Claude Code:** запусти субагентом как описано выше.
>
> **Gemini CLI:** субагент недоступен. Выполни inline: WebSearch по теме, синтезируй summary и список источников.
>
> **Codex:** если в текущей сессии доступен agent/subagent tool, можно делегировать исследование. Иначе сообщи пользователю `subagents unavailable in this session, continuing inline` и выполни ту же inline-проверку.

**Постобработка субагентов:** 1) отбросить НПА со status «отменен», 2) встроить реквизиты через речевые обороты из `speech-patterns.md`, 3) не вставлять сырой JSON в текст.

## Генерация .docx

```python
import sys
sys.path.insert(0, "skills/docs-memo")
from generate import create_sluzhebka

create_sluzhebka(
    body_paragraphs=[
        "Прошу Вас...",
    ],
    doc_date="18.02.2026",
    output_path="<output_dir_memo из ~/.docs-plugin/org_details.md>/Новая служебка.docx",
)
```

Параметры `addressee`, `sender`, `signer` загружаются из `~/.docs-plugin/org_details.md` автоматически, если не переданы явно.

## Политика вопросов

**Спрашивать:** суть и тема, если пользователь не указал.

**Не спрашивать** (есть дефолты из `~/.docs-plugin/org_details.md`):
- Адресат и автор
- Дата - по умолчанию сегодня
- Путь сохранения - из `output_dir_memo` в `~/.docs-plugin/org_details.md`

## Стиль текста

- **Кавычки:** только «елочки» (« »)
- **Обращение:** «Прошу Вас» с заглавной «В»
- **Никаких домыслов:** нет данных - спроси пользователя
- **Запрет косой черты:** не использовать «/» как разделитель слов. Альтернативы: запятая, «или», «и», скобки.

## Чек-лист

- [ ] Текст соответствует официально-деловому стилю
- [ ] Кавычки - только «елочки»
- [ ] Дата указана корректно
- [ ] Адресат и автор заполнены
- [ ] Файл сохранен в папку `output_dir_memo` из `~/.docs-plugin/org_details.md`
